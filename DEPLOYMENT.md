# Deployment runbook

Fastest-to-live stack: **Vercel** (web) · **Render** (API, Docker) · **Neon** (Postgres) · **Upstash** (Redis). Mostly free tiers.

Order matters: provision data stores → deploy API → deploy web → wire the URLs back together.

Prerequisites: the code pushed to a GitHub repo (you have one). The API Dockerfile is already validated locally (`docker build -f apps/api/Dockerfile .` + a from-scratch `migrate deploy`).

---

## 0. Push to GitHub

```bash
git remote add origin <your-repo-url>   # if not already added
git push -u origin main
```

## 1. Neon — Postgres

1. Create a project at https://neon.tech (region near your Render region, e.g. US West).
2. Copy the **pooled** connection string. Ensure it ends with `?sslmode=require`, e.g.
   `postgresql://USER:PASS@ep-xxx-pooler.us-west-2.aws.neon.tech/neondb?sslmode=require`
3. Keep it for `DATABASE_URL`. (Migrations run automatically when the API container boots.)

## 2. Upstash — Redis

1. Create a Redis database at https://upstash.com (same region if possible).
2. Copy the **`rediss://`** URL (TLS), e.g. `rediss://default:PASS@xxx.upstash.io:6379`.
3. Keep it for `REDIS_URL`. (ioredis speaks TLS automatically for `rediss://`.)

## 3. Render — API

1. https://render.com → **New** → **Blueprint** → connect the GitHub repo. Render reads `render.yaml`.
   - (Or **New → Web Service → Docker**, Dockerfile path `apps/api/Dockerfile`, context `.`.)
2. Set the `sync:false` env vars in the dashboard:
   - `DATABASE_URL` → Neon string (step 1)
   - `REDIS_URL` → Upstash URL (step 2)
   - `WEB_ORIGIN` → leave blank for now (fill after step 4)
   - `FINNHUB_API_KEY` / `COINGECKO_API_KEY` → optional (CoinGecko works without a key)
   - `JWT_SECRET` is auto-generated by Render.
3. Deploy. Watch the logs for `All migrations have been successfully applied` then `🚀 API is running`.
4. Copy the service URL, e.g. `https://investment-tracker-api.onrender.com`. Verify:
   `curl https://investment-tracker-api.onrender.com/api/health` → `{"status":"ok",...}`

> Free Render web services sleep after ~15 min idle; the first request then takes ~30s to wake.

## 4. Vercel — Web

1. https://vercel.com → **Add New → Project** → import the GitHub repo. Vercel reads `vercel.json`
   (build `pnpm nx build web`, output `dist/apps/web`, SPA rewrites).
2. Project **Settings → Environment Variables** (Production):
   - `VITE_API_URL` = `https://<render-app>.onrender.com/api`
   - `VITE_SOCKET_URL` = `https://<render-app>.onrender.com`
   (These are build-time for Vite — redeploy after changing them.)
3. Deploy. Copy the Vercel URL, e.g. `https://investment-tracker.vercel.app`.

## 5. Close the loop (CORS)

1. Back in Render, set `WEB_ORIGIN` = the Vercel URL (e.g. `https://investment-tracker.vercel.app`)
   and redeploy (or save — Render restarts).
2. Open the Vercel URL, register an account, create a portfolio, record a trade. Confirm valuation
   and history load and live prices stream.

---

## Continuous deploys

- Push to `main` → Render rebuilds the API (`autoDeploy: true`) and Vercel rebuilds the web app.
- `.github/workflows/ci.yml` runs lint/typecheck/test/build on PRs and `main`.

## Environment variables reference

| Where  | Var | Value |
|--------|-----|-------|
| Render | `DATABASE_URL` | Neon pooled string (`?sslmode=require`) |
| Render | `REDIS_URL` | Upstash `rediss://…` |
| Render | `JWT_SECRET` | auto-generated |
| Render | `WEB_ORIGIN` | Vercel URL |
| Render | `FINNHUB_API_KEY` / `COINGECKO_API_KEY` | optional |
| Vercel | `VITE_API_URL` | `https://<render>/api` |
| Vercel | `VITE_SOCKET_URL` | `https://<render>` |

## Troubleshooting

- **CORS errors in the browser** → `WEB_ORIGIN` on Render must exactly match the Vercel origin (no trailing slash).
- **Socket.io won't connect** → check `VITE_SOCKET_URL` (no `/api`, no trailing slash) and that `WEB_ORIGIN` is set.
- **DB connection errors** → use Neon's *pooled* host and keep `?sslmode=require`.
- **Migration didn't apply** → check Render boot logs; the container runs `prisma migrate deploy` before starting.
- **Slow first request** → free Render dynos sleep when idle; expected.
